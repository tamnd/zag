# v0.0.212.2

Patch: fix cross-compilation for all 5 targets (Linux musl, macOS, Windows).

## Changes

- `src/vm/signal_mod.zig`: use POSIX `sigemptyset`/`sigaddset`/`sigismember` for
  `pthread_sigmask` so `sigset_t` (an opaque array on Linux/musl) is never touched
  as an integer; guard Windows-incompatible calls (`strsignal`, `alarm`,
  `pthread_sigmask`) with early comptime returns.
- `src/vm/socket_mod.zig`: change `Fd = c.fd_t` → `c_int` (c.fd_t is `*anyopaque`
  on Windows); add `@ptrCast(@alignCast(...))` to five `bind`/`connect`/`sendto`
  calls (Linux sockaddr has alignment 2); wrap all POSIX-only socket syscalls and
  network helper externs (`inet_pton`, `ntohs`, etc.) in a `posix_net` conditional
  struct so Windows targets link cleanly.
- `src/vm/ssl_mod.zig`: same `Fd = c_int` fix; wrap `htons`/`inet_pton` externs in
  `posix_net`; guard `recvWithPending`, `sslSendall`, `sslSend`, `sslClose`,
  `sslShutdown`, `getServerCertificateFn` with Windows stubs.
- `src/vm/select_mod.zig`: change `c.fd_t` → `c_int` for `FdEntry.fd` and
  `valueToFd`; guard poll-using code path on Windows.
- `src/vm/selectors_mod.zig`: same `c_int` fixes; guard `selSelectKw` on Windows.
- `src/vm/mmap_mod.zig`: wrap POSIX-only externs (`mmap`, `munmap`, `msync`,
  `madvise`, `getpagesize`) in a `posix_mmap` conditional struct with Windows
  stubs; guard `entry.file.handle` (HANDLE on Windows) from being cast to `c_int`.
- `src/vm/subprocess_mod.zig`: guard `child.id` (process HANDLE on Windows) from
  being cast to integer pid.
