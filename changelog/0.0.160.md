# v0.0.160 — zipfile

Ship fixture 160: the `zipfile` module with full read/write support.

## What's in

- `ZipFile` read (`'r'`), write (`'w'`), append (`'a'`) modes
- STORED, DEFLATE, BZIP2, and LZMA compression on both read and write paths
- `writestr` with keyword `compress_type`; accepts a `ZipInfo` as first arg
- `namelist`, `infolist`, `getinfo`, `read`, `open`, `testzip`
- `extractall(path)` and `extract(name, path)` write files to disk
- `ZipInfo(filename)` with `is_dir()`, `filename`, `file_size`, `compress_type`, `compress_size`
- `is_zipfile(path)` — reads the file and checks for an EOCD signature
- `BadZipFile` (and `BadZipfile` alias) raised when opening a non-zip file in read mode
- Module constants: `ZIP_STORED`, `ZIP_DEFLATED`, `ZIP_BZIP2`, `ZIP_LZMA`

## Bug fixed in this release

Opening a garbage file in `'r'` mode previously succeeded silently because `parseZip` returned an empty list without error when no EOCD record was present. Added `hasValidZipEOCD` pre-check that raises `BadZipFile` before attempting to parse.
