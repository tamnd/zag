# 0.0.147 — 2026-04-26

### Fixed

- `fnmatch` bracket classes now match CPython for two edge cases:
  `]` as the first char of a class is literal (so `[]x]` is a valid
  class containing `]` and `x`), and unclosed brackets like `[!]c`
  fall back to a literal `[` instead of behaving as an empty
  negated class.
- `fnmatch.translate` emits `(?:...)` instead of `(?s:...)` so its
  output parses with the bundled `re` engine, which doesn't accept
  inline flag groups.
