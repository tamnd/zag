# 0.0.158 — 2026-04-27

The bz2 module. This is the first release that follows the new development
flow: fixture copied from goipy, spec written before implementation, PR
reviewed before merge.

## What changed

### Added bz2 module

The `bz2` module is the standard library's interface to the bzip2 compression
algorithm. It mirrors the shape of the `gzip` module that shipped in v0.0.157,
so if you already know how to use `gzip.open`, picking up `bz2.open` takes
about thirty seconds.

The one-shot functions are the most common path:

```python
import bz2
data = b"hello world " * 100
compressed = bz2.compress(data)
assert bz2.decompress(compressed) == data
```

`compress` accepts a `compresslevel` keyword argument (1 = fast, 9 = smallest,
default 9). `decompress` handles multi-member streams: two bz2-compressed blobs
concatenated together decompress cleanly into their concatenated contents.
Bad input raises `OSError`.

For incremental work, `BZ2Compressor` and `BZ2Decompressor` give you a
streaming interface:

```python
c = bz2.BZ2Compressor()
out = c.compress(b"hello ") + c.compress(b"world") + c.flush()

d = bz2.BZ2Decompressor()
result = d.decompress(out)
print(d.eof)         # True
print(d.unused_data) # b''
```

`BZ2Decompressor.decompress` accepts a `max_length` keyword argument that caps
the number of bytes returned in a single call. `needs_input` tells you whether
the decompressor needs more input bytes before it can produce output.

`BZ2File` wraps a bz2-compressed file with the standard file interface:
`read`, `write`, `readline`, `seek`, `tell`, `peek`, and context manager
support. The `name`, `mode`, `readable`, `writable`, and `closed` attributes
all behave as expected.

Finally, `bz2.open` opens a bz2-compressed file in binary or text mode,
matching the `gzip.open` signature including the `compresslevel` keyword
argument for write modes.

### Compatibility fixes

Two core fixtures were updated to match goipy exactly:

- **fixture 112 (readline):** `readline.backend` is now printed directly
  instead of through a membership test. The `read_init_file()` call is
  invoked without arguments, matching the CPython default.

- **fixture 143 (stat):** Added the macOS-specific stat constants that goipy
  tests: `S_IFWHT`, `S_ISWHT`, the `UF_*` user flag constants
  (`UF_COMPRESSED`, `UF_HIDDEN`, `UF_TRACKED`, `UF_DATAVAULT`,
  `UF_SETTABLE`), and the `SF_*` superuser flag constants (`SF_FIRMLINK`,
  `SF_RESTRICTED`, `SF_SUPPORTED`, `SF_DATALESS`, `SF_SYNTHETIC`,
  `SF_SETTABLE`).

### Test suite reorganization

Fixtures 158 through 260 have been moved from `tests/fixtures/` into a new
`tests/fixtures/ext/` subdirectory, renumbered 001 through 103. These are
the extra coverage fixtures that were written to explore existing features;
they do not correspond to individual module releases. The core fixture
sequence now cleanly tracks one fixture per module, starting from 001 and
continuing without gaps up to 157 (gzip) and now 158 (bz2).

The build system runs both suites: `zig build test` compiles and checks
`tests/fixtures/*.py` via `tests/integration.zig` and
`tests/fixtures/ext/*.py` via `tests/ext_integration.zig`.

### Changelog format

Starting with this release, each version gets its own file under
`changelog/`. The old monolithic `CHANGELOG.md` has been split into one
file per release. The format is prose-first rather than pure bullet lists,
so there is enough context to understand a change without needing to read
the diff.
