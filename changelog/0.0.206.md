# v0.0.206

Implement email module family (fixture 206).

## Changes

- `src/vm/email_mod.zig`: new module covering the full email package family:
  email (message_from_string), email.message (Message class with ordered
  case-insensitive duplicate-allowed headers, get_payload/set_payload,
  get_content_type/maintype/subtype/charset, get_param/params, is_multipart,
  attach, walk, as_string), email.mime.text (MIMEText with base64 encoding
  when charset given), email.mime.multipart (MIMEMultipart), email.mime.application
  (MIMEApplication), email.utils (parseaddr, formataddr, formatdate, make_msgid),
  email.header (decode_header with base64 encoded-word support, make_header),
  email.encoders (encode_noop, encode_base64), email.errors (MessageError,
  MessageParseError, HeaderParseError, MultipartConversionError), email.generator
  (Generator with flatten), email.parser (stub); sub-modules exposed as
  attributes on parent modules so dotted imports resolve correctly.
- `src/vm/interp.zig`: add email_module, email_message_module, email_mime_*_module,
  email_utils_module, email_header_module, email_encoders_module, email_errors_module,
  email_generator_module, email_parser_module, email_message_class,
  email_header_class, email_generator_class fields; register all email sub-modules
  in getBuiltinModule.
- `tests/fixtures/206_*`: fixture from goipy.
